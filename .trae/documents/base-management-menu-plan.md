# 基础管理菜单增加计划

## 项目结构分析

当前项目是一个担保集团业务财务系统，使用 Spring Boot + Vue 3 技术栈。前端路由配置位于 `guarantee-finance-frontend/src/router/index.ts` 文件中。

### 当前菜单结构
- 仪表盘
- 基础信息同步
- 收款单管理
- 付款单管理
- 银行对账
- 银企直连
- 会计平台
- CFCA认证
- 定时任务
- 机构管理（一级菜单）
- 用户管理（一级菜单）
- 角色管理（一级菜单）
- 菜单权限（一级菜单）
- 审批流程（一级菜单）

## 需求分析

用户要求：
- 增加基础管理菜单作为一级菜单
- 将机构管理、用户管理、角色管理、菜单权限放在基础管理的二级菜单下

## 实现方案

### 1. 修改前端路由配置

**文件：** `guarantee-finance-frontend/src/router/index.ts`

**修改内容：**
- 添加基础管理一级菜单
- 将机构管理、用户管理、角色管理、菜单权限移动到基础管理下作为二级菜单
- 保持审批流程为独立的一级菜单

### 2. 调整路由路径

**修改方案：**
- 基础管理一级菜单路径：`/system/base`
- 机构管理：`/system/base/org`
- 用户管理：`/system/base/user`
- 角色管理：`/system/base/role`
- 菜单权限：`/system/base/menu`
- 审批流程保持：`/system/process`

### 3. 调整前端组件路径

**检查并确认：**
- 前端组件文件是否需要调整路径
- 确保组件引用正确

### 4. 后端API路径检查

**检查内容：**
- 后端API路径是否与新的前端路由路径匹配
- 确保所有系统管理相关的API接口正常工作

## 实施步骤

1. **修改前端路由配置**
   - 在路由配置中添加基础管理一级菜单
   - 移动相关管理模块到二级菜单
   - 调整路由路径和组件引用

2. **验证前端页面**
   - 确保基础管理菜单显示正确
   - 确保二级菜单点击正常
   - 确保页面功能正常

3. **验证后端API**
   - 测试机构管理、用户管理、角色管理、菜单权限的API接口
   - 确保所有接口正常响应

4. **测试整体功能**
   - 测试登录流程
   - 测试基础管理下的所有功能
   - 测试其他模块功能不受影响

## 风险评估

### 潜在风险
1. **路由路径冲突**：修改路由路径可能导致与现有路径冲突
2. **组件引用错误**：调整路径后可能导致组件引用失败
3. **API接口调用失败**：路径变更可能影响API调用
4. **权限管理问题**：菜单结构变更可能影响权限控制

### 风险缓解措施
1. **备份当前路由配置**：在修改前备份原始文件
2. **逐步修改**：先修改路由配置，再测试功能
3. **全面测试**：修改后测试所有相关功能
4. **检查权限**：确保权限管理不受影响

## 技术要点

- **Vue Router**：使用嵌套路由实现二级菜单
- **路径调整**：保持后端API路径不变，仅调整前端路由路径
- **组件引用**：确保组件路径引用正确
- **权限控制**：确保权限管理逻辑不受影响
