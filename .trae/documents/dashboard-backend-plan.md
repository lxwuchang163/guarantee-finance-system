# 仪表盘后台功能模块开发和数据设计计划

## 1. 前端功能分析

### 1.1 统计卡片功能
- **今日收款**：显示当日收款金额及环比变化
- **今日付款**：显示当日付款金额及环比变化
- **待审核单据**：显示待审核的单据数量及环比变化
- **本月对账**：显示本月对账数量及完成率

### 1.2 待办事项功能
- 显示待处理和已完成的任务
- 包含事项名称、类型、优先级、创建时间、状态
- 支持处理和查看操作

### 1.3 系统公告功能
- 显示系统通知和公告
- 支持点击查看公告详情
- 包含公告标题、发布时间、内容

### 1.4 业务概览图表功能
- **收支趋势**：显示月度收支数据的柱状图
- **业务类型分布**：显示不同业务类型的占比饼图

## 2. 后台接口设计

### 2.1 统计数据接口
- **GET /api/dashboard/stats**：获取仪表盘统计数据
  - 响应：包含今日收款、今日付款、待审核单据、本月对账等统计数据

### 2.2 待办事项接口
- **GET /api/dashboard/todos**：获取待办事项列表
  - 响应：包含待办事项的详细信息
- **POST /api/dashboard/todos/{id}/process**：处理待办事项
  - 响应：处理结果
- **GET /api/dashboard/todos/{id}**：获取待办事项详情
  - 响应：待办事项的详细信息

### 2.3 系统公告接口
- **GET /api/dashboard/notices**：获取系统公告列表
  - 响应：包含公告的基本信息
- **GET /api/dashboard/notices/{id}**：获取公告详情
  - 响应：公告的详细信息

### 2.4 业务数据接口
- **GET /api/dashboard/business/income-expense**：获取收支趋势数据
  - 响应：包含月度收支数据
- **GET /api/dashboard/business/type-distribution**：获取业务类型分布数据
  - 响应：包含不同业务类型的占比数据

## 3. 数据模型设计

### 3.1 统计数据模型
- **DashboardStats**：仪表盘统计数据
  - todayIncome：今日收款
  - todayExpense：今日付款
  - pendingDocuments：待审核单据
  - monthlyReconciliation：本月对账
  - changeRates：环比变化率

### 3.2 待办事项模型
- **TodoItem**：待办事项
  - id：唯一标识
  - title：事项名称
  - type：类型
  - priority：优先级
  - createTime：创建时间
  - status：状态（待处理/已完成）
  - description：描述

### 3.3 系统公告模型
- **Notice**：系统公告
  - id：唯一标识
  - title：公告标题
  - content：公告内容
  - publishTime：发布时间
  - status：状态

### 3.4 业务数据模型
- **IncomeExpenseData**：收支数据
  - month：月份
  - income：收入
  - expense：支出

- **BusinessTypeData**：业务类型数据
  - type：业务类型
  - value：占比
  - percentage：百分比

## 4. 后台功能实现计划

### 4.1 开发步骤
1. **创建数据模型**：定义上述数据模型
2. **创建Mapper接口**：实现数据访问层
3. **创建Service接口**：实现业务逻辑
4. **创建Controller**：实现API接口
5. **添加配置**：添加必要的配置和依赖
6. **测试接口**：测试所有接口的功能
7. **修复bug**：修复测试中发现的问题

### 4.2 技术栈
- **后端框架**：Spring Boot 3.2.5
- **ORM框架**：MyBatis-Plus
- **数据库**：MySQL
- **API风格**：RESTful
