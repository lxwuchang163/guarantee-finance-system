# 担保集团业务财务系统规格说明

## Why

担保集团需要一套完整的业务财务一体化系统，实现业务数据与用友NC Cloud财务系统的无缝对接，涵盖基础信息同步、收付款单处理、凭证管理、银企直连、CFCA认证等核心功能，提升财务管理效率和数据准确性。

## What Changes

- 新建担保集团业务财务系统，包含11大功能模块
- 实现与用友NC Cloud财务系统的双向数据交互
- 集成银企直连和CFCA数字证书支付认证能力
- 提供定时任务调度、银行对账、会计平台等完整财务处理流程

## Impact

- 涉及系统：前端Web应用、后端API服务、数据库、用友NC Cloud接口、银行接口、CFCA认证服务
- 技术栈：Spring Boot + Vue3 + MySQL + Redis + RabbitMQ/Kafka

## ADDED Requirements

### Requirement 1: 基础信息同步模块

#### 1.1 客户信息同步

- **功能描述**: 从业务系统同步客户基本信息到本系统及用友NC Cloud
- **同步字段**:
  - 客户编码（唯一标识）
  - 客户名称（全称/简称）
  - 客户类型（个人/企业/机构）
  - 统一社会信用代码/身份证号
  - 联系电话、联系人
  - 注册地址、经营地址
  - 所属行业、客户等级
  - 创建时间、更新时间
- **同步方式**:
  - 全量同步：首次初始化或手动触发全量拉取
  - 增量同步：基于时间戳或变更标记的增量更新
  - 实时同步：通过消息队列接收业务系统的变更通知
- **数据处理规则**:
  - 新增客户：直接插入
  - 变更客户：比对差异字段，记录变更日志
  - 停用客户：标记状态，保留历史数据
  - 冲突处理：以业务系统为准，记录冲突日志
- **接口要求**:
  - 支持分页查询（默认每页1000条）
  - 支持按最后修改时间增量查询
  - 支持按客户编码批量查询
  - 同步结果需返回成功/失败数量及失败原因

#### 1.2 业务品种同步

- **功能描述**: 同步担保业务的品种分类信息
- **同步字段**:
  - 品种编码、品种名称
  - 品种父级编码（支持树形结构）
  - 品种类型（融资担保/非融资担保/其他）
  - 费率标准（基准费率、最低费率、最高费率）
  - 期限范围（最短期限、最长期限）
  - 担保方式（一般保证/连带责任保证/抵押/质押等）
  - 是否启用、排序号
- **同步规则**:
  - 品种树形结构需保持层级关系完整
  - 已关联业务的品种不允许删除，仅允许停用
  - 费率变更需记录历史版本

#### 1.3 客户银行账号同步

- **功能描述**: 同步客户的银行账户信息
- **同步字段**:
  - 账号ID、客户编码（关联客户）
  - 银行名称、开户行名称
  - 银行账号、联行号
  - 账户性质（基本户/一般户/专用户/临时户）
  - 是否默认账户、是否主结算账户
  - 账户状态（正常/冻结/销户）
  - 开户日期、销户日期
- **特殊处理**:
  - 同一客户可维护多个银行账户
  - 默认账户唯一性校验
  - 账号脱敏显示（中间4位隐藏）

#### 1.4 机构信息同步

- **功能描述**: 同步集团及下属分支机构信息
- **同步字段**:
  - 机构编码、机构名称
  - 机构类型（集团总部/分公司/子公司/办事处）
  - 上级机构编码（树形结构）
  - 所在地区（省/市/区县）
  - 详细地址、联系电话
  - 负责人姓名、负责人联系方式
  - 组织机构代码证号
  - 是否独立核算、是否启用
- **同步规则**:
  - 机构树形结构需保持完整
  - 下级机构的上级机构必须存在
  - 已有关联数据的机构仅允许停用不允许删除

#### 1.5 基础信息同步通用功能

- **同步任务管理**:
  - 手动触发同步（立即执行）
  - 定时自动同步（可配置执行周期）
  - 同步任务状态跟踪（待执行/执行中/已完成/失败）
  - 同步日志记录（执行时间、耗时、处理条数、错误详情）
- **数据校验规则**:
  - 必填字段非空校验
  - 编码唯一性校验
  - 关联数据存在性校验
  - 字段格式校验（手机号、身份证、统一社会信用代码等）
  - 枚举值合法性校验
- **异常处理**:
  - 单条数据失败不影响整体同步
  - 失败数据记录到异常表，支持重试
  - 异常数据告警通知（邮件/短信/站内信）

### Requirement 2: 收款单处理模块

#### 2.1 保费收入业务

- **功能描述**: 处理担保业务收取的保费收入
- **业务字段**:
  - 收款单号（系统自动生成，规则：SK+年月日+序号）
  - 业务类型：保费收入
  - 客户编码、客户名称
  - 合同编号、项目名称
  - 业务品种、币种
  - 收款金额（大写/小写）
  - 收款日期、实际到账日期
  - 收款方式（转账/现金/支票/汇票/其他）
  - 付款方名称、付款方账号
  - 付款方开户行、收款方账号
  - 收款方开户行、用途/摘要
  - 凭证号（关联会计凭证）
  - 状态（草稿/已审核/已记账/已作废）
  - 制单人、审核人、记账人
  - 制单时间、审核时间、记账时间
  - 附件（支持多文件上传，PDF/图片/Excel）
- **业务规则**:
  - 收款金额必须大于0
  - 收款日期不能晚于当前日期
  - 同一合同下保费收款累计不超过合同约定金额
  - 已记账的收款单不允许修改/删除，仅允许红字冲销
  - 支持批量导入收款单（Excel模板）

#### 2.2 分担收入业务

- **功能描述**: 处理再担保或其他担保机构分担的收入款项
- **特有字段**:
  - 分担方编码、分担方名称
  - 分担比例（%）、原担保金额
  - 分担金额、剩余自留金额
  - 分担协议编号
  - 分担类型（再担保分入/联合担保分成/其他）
- **业务规则**:
  - 分担金额 = 原担保金额 × 分担比例
  - 分担比例需在0-100%之间
  - 需关联对应的原担保合同

#### 2.3 追偿到款收入业务

- **功能描述**: 处理代偿后向被担保人或反担保人追偿收回的款项
- **特有字段**:
  - 代偿单号（关联原代偿记录）
  - 追偿对象编码、追偿对象名称
  - 追偿方式（协商还款/诉讼执行/资产处置/其他）
  - 本次追偿金额、累计追偿金额
  - 代偿本金余额、代偿余额
  - 追偿费用（律师费/诉讼费/评估费/其他）
- **业务规则**:
  - 必须关联有效的代偿记录
  - 累计追偿金额不超过代偿金额+追偿费用上限
  - 追偿完成后更新代偿记录状态

#### 2.4 收款单通用功能

- **单据操作**:
  - 新增、修改、删除（草稿状态）
  - 提交审核、审核通过/驳回
  - 记账、冲销
  - 打印（支持套打模板）
  - 导出（Excel/PDF）
- **审批流程**:
  - 可配置多级审批流
  - 支持加签、转签、驳回
  - 审批意见记录
  - 审批超时提醒
- **查询统计**:
  - 多条件组合查询（客户/日期范围/金额范围/业务类型/状态）
  - 收款汇总表（按日/周/月/季度/年）
  - 客户收款明细
  - 业务品种收款分析
  - 未核销收款清单

### Requirement 3: 付款单处理模块

#### 3.1 退费支出业务

- **功能描述**: 处理担保业务退还给客户的保费
- **业务字段**:
  - 付款单号（FK+年月日+序号）
  - 业务类型：退费支出
  - 客户编码、客户名称
  - 合同编号、原收款单号（关联原收款）
  - 退费原因（提前解保/部分解除/费率调整/其他）
  - 退费金额、原收款金额
  - 退费比例（%）、计算说明
  - 收款方名称、收款方账号
  - 收款方开户行、付款方账号
  - 付款方开户行、用途/摘要
  - 状态（草稿/已审核/已付款/已记账/已作废）
- **业务规则**:
  - 退费金额不超过原实收金额
  - 必须关联原收款单
  - 退费后合同保费余额不能为负数
  - 大额退费需走特别审批流程

#### 3.2 代偿支出业务

- **功能描述**: 处理履行担保责任向债权人支付的代偿款
- **特有字段**:
  - 担保合同编号、债权人称谓
  - 债权人账号、债权人开户行
  - 代偿本金、代偿利息
  - 违约金、其他费用
  - 代偿总金额
  - 代偿依据（判决书/调解书/和解协议/其他）
  - 代偿日期、追偿截止日期
  - 反担保物信息（如有）
- **业务规则**:
  - 代偿前需确认担保责任成立
  - 代偿金额需经严格审批
  - 代偿后自动生成追偿登记
  - 记录反担保物处置状态

#### 3.3 追回资金分配支出业务

- **功能描述**: 将追偿收回的资金按约定进行分配
- **特有字段**:
  - 追回资金总额
  - 分配方案：
    - 归还代偿本金金额
    - 支付追偿费用金额
    - 归还垫付利息金额
    - 其他分配项
  - 各分配项占比（%）
  - 分配受益方信息
  - 分配协议/决议编号
- **业务规则**:
  - 分配总额 = 追回资金总额
  - 各分配项之和等于追回资金总额
  - 分配方案需经审批确认
  - 分配完成后更新各相关科目余额

#### 3.4 付款单通用功能

- **单据操作**: 与收款单类似的新增/修改/删除/审核/记账/打印/导出
- **支付集成**:
  - 对接银企直连发起支付指令
  - 支付状态实时跟踪
  - 支付失败自动重试（可配置次数）
  - 支付结果回写付款单
- **付款控制**:
  - 单笔限额控制
  - 日累计限额控制
  - 账户余额校验
  - 大额支付双人复核
  - 支付白名单管理

### Requirement 4: 客户档案同步模块（用友NC Cloud对接）

#### 4.1 功能概述

- **目标**: 将本系统维护的客户档案信息同步至用友NC Cloud财务系统
- **同步内容**:
  - 客户档案（客户编码、客户名称、客户简称、客户类型等）
  - 客户分类（按行业、地区、规模等维度）
  - 客户扩展属性（自定义字段）
  - 客户银行账户信息

#### 4.2 映射关系

| 本系统字段                 | 用友NC Cloud字段 | 说明       |
| --------------------- | ------------ | -------- |
| customer\_code        | pk\_customer | 客户主键     |
| customer\_name        | name         | 客户名称     |
| customer\_short\_name | shortname    | 客户简称     |
| customer\_type        | custtype     | 客户类型     |
| credit\_code          | defitem1     | 统一社会信用代码 |
| industry              | indclass     | 所属行业     |
| contact\_phone        | phone        | 联系电话     |
| address               | address      | 地址       |
| is\_enabled           | isenable     | 是否启用     |

#### 4.3 同步策略

- **实时同步**: 客户信息在本系统保存/修改时立即同步
- **批量同步**: 支持手动选择批量客户进行同步
- **差异比对**: 同步前比对双方数据差异，仅同步变更字段
- **冲突解决**: 以本系统为准或以NC Cloud为准（可配置）
- **同步日志**: 记录每次同步的时间、内容、结果

#### 4.4 异常处理

- NC Cloud接口调用失败自动重试（指数退避策略）
- 数据格式转换失败记录详细错误信息
- 网络超时处理（配置超时时间和重试次数）
- 同步失败数据进入异常队列，支持手动重新同步

### Requirement 5: 凭证数据同步模块（用友NC Cloud对接）

#### 5.1 功能概述

- **目标**: 将本系统生成的会计凭证同步至用友NC Cloud财务系统
- **同步内容**:
  - 凭证头信息（凭证号、凭证日期、制单人、审核人等）
  - 凭证明细（科目、借贷方向、金额、摘要、辅助核算等）
  - 凭证附件信息

#### 5.2 凭证数据结构

```
凭证头(Voucher):
  - voucher_id: 凭证ID
  - voucher_no: 凭证编号
  - voucher_date: 凭证日期
  - voucher_type: 凭证类型（收/付/转）
  - accounting_period: 会计期间
  - company_code: 公司编码
  - maker: 制单人
  - auditor: 审核人
  - poster: 记账人
  - attachment_count: 附件张数
  - total_debit: 借方合计
  - total_credit: 贷方合计
  - status: 状态（未审核/已审核/已记账/已作废）
  - remark: 备注
  - source_bill_type: 来源单据类型
  - source_bill_no: 来源单据号

凭证明细(VoucherDetail):
  - detail_id: 明细ID
  - voucher_id: 凭证ID（外键）
  - line_no: 行号
  - account_code: 科目编码
  - account_name: 科目名称
  - direction: 方向（借/贷）
  - amount: 金额
  - summary: 摘要
  - auxiliary_accounting: 辅助核算信息
    - customer_code: 客户编码
    - department_code: 部门编码
    - project_code: 项目编码
    - supplier_code: 供应商编码
    - employee_code: 经办人编码
  - debit_amount: 借方金额
  - credit_amount: 贷方金额
```

#### 5.3 凭证映射规则

| 业务场景   | 借方科目       | 贷方科目          | 金额来源 |
| ------ | ---------- | ------------- | ---- |
| 保费收入   | 银行存款/应收保费  | 保费收入          | 收款金额 |
| 分担收入   | 银行存款       | 分担收入          | 分担金额 |
| 追偿到款   | 银行存款/现金    | 追偿收入/代偿支出(红字) | 到款金额 |
| 退费支出   | 保费收入(红字)   | 银行存款          | 退费金额 |
| 代偿支出   | 代偿支出/应收代偿款 | 银行存款          | 代偿金额 |
| 追回资金分配 | 各分配对应科目    | 银行存款/代偿往来     | 分配金额 |

#### 5.4 同步流程

1. 本系统生成凭证 → 凭证暂存（本地）
2. 凭证审核 → 标记为"待同步"
3. 触发同步 → 调用NC Cloud接口
4. NC Cloud返回结果 → 更新同步状态
5. 同步成功 → 标记为"已同步"
6. 同步失败 → 进入重试队列

#### 5.5 凭证管理功能

- **凭证查询**: 多条件查询、凭证预览
- **凭证修改**: 未审核凭证可修改，已审核凭证需冲销后重做
- **凭证审核**: 支持批量审核
- **凭证记账**: 同步至NC Cloud后由NC Cloud完成记账

### Requirement 6: 银行对账模块

#### 6.1 功能概述

- **目标**: 将收付款单据与银行流水进行自动核对，发现差异并处理
- **对账范围**: 所有通过银企直连获取的银行流水 vs 系统内收付款单

#### 6.2 银行流水管理

- **流水获取方式**:
  - 银企直连自动下载（定时任务）
  - 手动导入（Excel/银行标准格式）
  - API实时推送（如银行支持）
- **流水字段**:
  - 流水号、交易日期、记账日期
  - 交易类型（收入/支出/其他）
  - 收方账号、收方名称、收方开户行
  - 付方账号、付方名称、付方开户行
  - 交易金额、币种、汇率
  - 摘要/用途、备注
  - 银行凭证号
  - 对账状态（未匹配/已匹配/异常）

#### 6.3 自动对账规则

- **精确匹配条件**（全部满足）:
  - 交易方向一致（收/付）
  - 金额完全一致
  - 交易日期在对账容差范围内（默认±3天）
  - 对方账号一致
- **模糊匹配条件**（满足其中多项）:
  - 金额一致 + 交易日期接近 + 对方名称相似
  - 金额一致 + 摘要关键字匹配
  - 金额一致 + 参考号/备注匹配
- **匹配优先级**: 精确匹配 > 模糊匹配 > 人工确认

#### 6.4 对账结果处理

- **自动勾兑**: 匹配度100%的单据自动勾兑
- **待确认列表**: 模糊匹配的单据需人工确认
- **异常列表**: 无法匹配或金额不符的单据
- **差异类型**:
  - 金额差异：银行流水金额 ≠ 单据金额
  - 方向差异：银行方向 ≠ 单据方向
  - 时间差异：超出容差范围
  - 未达账项：一方有记录另一方无记录
  - 重复记录：可能重复记账
- **差异处理**:
  - 生成银行余额调节表
  - 差异原因标注
  - 差异调整建议
  - 支持手工强制勾兑（需授权）

#### 6.5 对账报表

- **银行存款余额调节表**
- **未达账项清单**
- **对账差异报告**
- **对账统计分析**（按账户/期间/差异类型）

### Requirement 8: CFCA支付认证模块

#### 8.1 功能概述

- **目标**: 集成中国金融认证中心(CFCA)数字证书认证服务，确保大额支付的安全性
- **应用场景**: 所有通过银企直连发起的大额支付指令

#### 8.2 数字证书管理

- **证书类型**:
  - 企业证书：企业级数字证书
  - 操作员证书：每个操作人员独立的数字证书
  - API证书：系统间调用的服务器证书
- **证书生命周期管理**:
  - 证书申请：在线申请或柜面申请
  - 证书下载：安全通道下载
  - 证书安装：导入证书库
  - 证书更新：到期前自动续期提醒
  - 证书吊销：证书泄露或人员离职时吊销
  - 证书有效期监控：临近过期告警（提前30/15/7天）

#### 8.3 签名验证流程

```
支付请求 → 数据组装 → CFCA签名 → 发送银行 → 银行验签 → 执行支付
                                                    ↓
响应结果 ← 数据解析 ← CFCA验签 ← 接收响应 ← 银行签名
```

#### 8.4 支付认证流程

1. **支付指令准备**: 组装支付数据
2. **数据签名**: 使用CFCA私钥对支付数据进行数字签名
3. **签名上传**: 将签名后的数据发送至CFCA认证服务器
4. **签名验证**: CFCA服务器验证签名有效性
5. **令牌获取**: 验证通过后获取支付令牌
6. **指令提交**: 携带支付令牌提交至银行
7. **结果确认**: 银行处理后返回结果

#### 8.5 安全控制措施

- **双因素认证**: 数字密码 + 硬件Key（UKey）
- **额度分级**:
  - ≤5万元：单人签名即可
  - 5万-50万元：需双人签名
  - <br />
    > 50万元：需三人签名+审批
- **签名审计**: 记录每次签名操作的详细信息
- **异常检测**:
  - 异常时间操作检测（非工作时间）
  - 异常地点操作检测（IP地址异常）
  - 异常金额检测（超过历史模式）
  - 短时间内多次签名尝试检测

- **功能描述**: 维护不同业务场景的凭证生成模板
- **模板要素**:
  - 模板编码、模板名称
  - 适用业务类型（保费收入/分担收入/追偿到款/退费/代偿/追回分配）
  - 凭证类型（收款凭证/付款凭证/转账凭证）

- **模板示例**（保费收入凭证模板）:

```

    辅助核算：[客户:{客户编码}]
贷：保费收入                     {保费金额}
    摘要：收到{客户名称}{合同编号}保费
```

#### 10.3 单据生成凭证

- **触发方式**:
  - 手动触发生成：用户选择单据后手动生成凭证

  - 批量生成：选择多张同类型单据批量生成
- **生成逻辑**:
  1. 读取业务单据数据
  2. 匹配合适的凭证模板
  3. 按模板规则填充凭证数据
  4. 校验凭证平衡（借=贷）
  5. 保存凭证（暂存态）
  6. 生成凭证与源单据的关联关系
- **生成规则**:
  - 一张收/付款单生成一张凭证
  - 一张凭证可包含多条分录
  - 支持合并生成（同一客户同一科目的多张单据合并）
  - 支持拆分生成（一张单据按不同科目拆分）

#### 10.4 凭证管理

- **凭证列表**: 展示所有凭证，支持多维筛选
- **凭证查看**: 凭证详情展示（含分录、辅助核算、附件）
- **凭证编辑**: 编辑未审核凭证的分录信息
- **凭证删除**: 删除未审核且未同步的凭证
- **凭证审核**: 单张审核/批量审核
- **凭证反审核**: 反审核已审核但未记账的凭证

  - 按凭证号查询

- **缓存**: Redis 7.x（缓存、分布式锁、Session共享）
- **消息队列**: RabbitMQ 或 Kafka（异步处理、事件驱动）
- **任务调度**: XXL-JOB 或 PowerJob（分布式任务调度）
- **接口文档**: Swagger/OpenAPI 3.0
- **日志**: SLF4J + Logback + ELK（日志收集分析）
- **监控**: Spring Boot Actuator + Prometheus + Grafana

### 前端技术栈

- **框架**: Vue 3 + TypeScript + Vite
- **UI组件库**: Element Plus / Ant Design Vue
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **HTTP客户端**: Axios
- **图表**: ECharts（数据可视化）
- **表格**: VxeTable（复杂表格场景）

### 安全要求

- **认证**: JWT Token + Refresh Token 双Token机制
- **授权**: Spring Security + 自定义RBAC
- **密码**: BCrypt加密存储
- **传输**: HTTPS 全链路加密
- **接口安全**:
  - 接口签名验证
  - 防重放攻击（Nonce + Timestamp）
  - 接口限流（令牌桶算法）
  - SQL注入防护（参数化查询）
  - XSS攻击防护（输入过滤+输出编码）
- **数据安全**:
  - 敏感数据加密存储（AES-256）
  - 数据脱敏展示
  - 操作日志审计
  - 数据备份恢复

### 性能要求

- **响应时间**:
  - 简单查询 < 200ms
  - 复杂查询 < 500ms
  - 单据保存 < 1s
  - 批量同步（1000条）< 10s
- **并发能力**:
  - 支持100+用户同时在线
  - 支持50+并发操作
- **数据量**:
  - 支持500万+业务单据
  - 支持100万+客户档案
  - 支持1000万+凭证分录

### 部署要求

- **容器化**: Docker + Docker Compose / Kubernetes
- **反向代理**: Nginx
- **负载均衡**: Nginx Upstream / 云厂商LB
- **数据库**: MySQL主从复制 + 读写分离
- **缓存**: Redis Cluster / Sentinel
- **文件存储**: MinIO / OSS（附件存储）
